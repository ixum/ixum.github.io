Python调试

给出python-pdb调试的方法和相关命令。

1. 什么是pdb

pdb是Python自带的一个包，主要是为Python提供一种交互的源码调试功能，类似于gdb。主要功能包括设置断点、单步调试、进入函数调试、查看当前代码、查看栈片段、动态改变变量的值等。常用的调试命令可以在下表中查询。

命令 简写命令 解释
args a 打印当前函数的参数
break b 设置断点
clear cl 清除断点
condition 无 设置条件断点
continue c或者cont 继续运行，知道遇到断点或者脚本结束
disable 无 禁用断点
enable 无 启用断点
help h 查看pdb帮助
ignore 无 忽略断点
jump j 跳转到指定行数运行
list l 列出脚本清单
next n 执行下条语句，遇到函数不进入其内部
p p 打印变量值，也可以用print
quit q 退出 pdb
return r 一直运行到函数返回
tbreak 无 设置临时断点，断点只中断一次
step s 执行下一条语句，遇到函数进入其内部
where w 查看所在的位置
! 无 在pdb中执行语句

2. 如何使用pdb

使用pdb调试有两种方式：

  1. 在源文件中引入pdb，并设置追踪锚点
  2. 正常编写源文件，使用Python -m pdb demo.py来启动调试。

下面给出一个案例。

2.1 引入追踪锚点

# !/bin/python
# -*- coding:utf-8 -*-
# filename: demo.py

import pdb
a = "This is "
pdb.set_trace()
b = "my first using "
c = "pdb."
line = a + b + c
print(line)

编写完成源码之后，在终端执行：

python demo.py

之后，在终端中输出了如下信息。

# mx @ debian in ~/workspace/python [9:26:29] 
$ python demo.py 
> /home/mx/workspace/python/demo.py(8)<module>()
-> b = "my first using "
(Pdb)

用户在Pdb后面输入相关的命令，就可以进行代码的调试。

开始调试：直接运行脚本，会停留在 pdb.set_trace() 处，选择 n+enter 可以执行当前的 statement。在第一次按下了 n+enter 之后可以直接按 enter 表示重复执行上一条 debug 命令。

2.2 命令启动调试

编写源文件。

# !/bin/python
# -*- coding:utf-8 -*-
# filename: demo.py

a = "This is "
b = "my first using "
c = "pdb."
line = a + b + c
print(line)

执行下列命令：

python -m pdb demo.py

也会输出如下的信息，这时就可以进行代码调试。

python
